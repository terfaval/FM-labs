# Lumira modal patch – content export

## Purpose

This patch provides the full Hungarian copy for the redesigned Lumira project modal.

It is intended for implementation inside the portfolio project modal as a screen-driven narrative walkthrough.

## General instructions for Codex

- Keep the modal linear and calm.
- The screenshots should lead the reading flow.
- Use short text blocks.
- Keep all text blocks center-aligned except the hero body if layout balance requires left alignment on large screens.
- Alternate screenshot placement: if one section has image on the right, the next image section should place it on the left.
- Do not paraphrase the copy unless necessary for layout fit.
- Preserve the tone: reflective, clear, restrained, not mystical, not overly therapeutic.
- Avoid overusing words like „rendszer”, „eszköz”, „platform”. Prefer „felület”, „folyamat”, „tér”, „munka”.
- Do not make stronger claims than the product currently supports.
- The app should feel credible, grounded, and intentionally non-dogmatic.
- Long paragraphs are only allowed in the intro and outro style blocks.
- Mood blocks should remain short, centered, and visually lighter.
- The “Működési elv” section must be an icon grid with short labels and concise supporting text.
- The “Következő irányok” section must have a distinct layout from the “Működési elv” grid. Use directional or roadmap-like cards, not the same icon grid pattern.

## Image alternation rule

For all image-based sections:
- alternate image side each time
- on mobile, stack vertically with image first
- keep the screenshot large enough to remain meaningful
- maintain consistent spacing between screenshot and copy

Suggested desktop rhythm:
- Block 3: text left / image right
- Block 4: image left / text right
- Block 6: text left / image right
- Block 8: image left / text right
- Block 9: text left / image right

## Final tone check for Codex

Before implementation, validate the following:
- The modal should feel curated, not content-dumped.
- The copy should feel grounded and believable.
- The product should read as thoughtful and real, not speculative.
- The visuals should carry at least half of the storytelling weight.
